# Security Policy

## Supported Versions
Only the current proprietary branch (`proprietary-main`) is supported.

Any versions of HashVektor, formerly known as CrunchHash, released previously under the MIT License are
no longer maintained and receive no security updates.

## Reporting a Vulnerability
If you discover a security issue, please report it privately.

Contact:
**Dustin Wayne Deen — PlanetArchives.org — support@planetarchives.org**

Please include:
- A clear description of the issue
- Steps to reproduce
- Potential impact
- Any suggested fixes

You will receive a response within a reasonable timeframe.

## Disclosure Policy
Do not disclose security vulnerabilities publicly until:
- The issue has been acknowledged, and
- A fix or mitigation is available

Unauthorized public disclosure may result in removal of issues or pull
requests that expose sensitive information.

## Scope
This policy applies only to:
- The proprietary HashVektor v1.0/v1.1
- Future commercial releases
- The portable version
- The MAUI rewrite

It does NOT apply to:
- Old MIT-licensed snapshots
- Forks created during the MIT period
- Third-party redistributions
